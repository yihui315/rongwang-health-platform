# 一辉智能体 Token 节省指南 v4.1

## 核心原则：Claude CEO 只做决策，不做内容生产

### 零Claude消耗的操作（直接委托）

| 任务类型 | 走哪个模型 | Claude消耗 |
|---------|-----------|-----------|
| SEO文章/博客/着陆页 | GPT-5.4 (Copilot) | 0 |
| 翻译/邮件/摘要 | GPT-4o (Copilot) | 0 |
| 小红书/微信/抖音 | MiniMax 2.7 | 0 |
| HTML/CSS/JS组件 | Qwen3 (本地) | 0 |
| 复杂代码 | Codex (Copilot) | 0 |
| 竞品分析 | Gemini (Copilot) | 0 |

### 需要少量Claude token的操作

| 操作 | 预估消耗 |
|-----|---------|
| 路由决策 | ~200 tokens |
| 任务分析 | ~500 tokens |
| 结果汇总 | ~300 tokens |
| 流水线编排 | ~800 tokens |

### 自动降级策略

- **正常模式** (0-70%): Claude做完整分析+路由
- **节省模式** (70-90%): 跳过分析，直接路由
- **极限模式** (90%+): 只做最基本路由

### 缓存机制

相同的 模型+prompt+system 组合会被缓存24小时。第二次调用零消耗。

### 使用建议

1. **用 `smart_route` 而非直接对话** — 自动选最优模型
2. **用 `token_status` 检查预算** — 知道还能做多少
3. **用 `token_optimize` 预评估** — 先看消耗再决定
4. **批量任务用 `multi_ai`** — 一次编排多个模型并行
5. **避免让Claude生成长文** — 那是GPT-5.4/MiniMax的活
