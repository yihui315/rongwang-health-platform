## 二、GEO（生成引擎优化）— AI 搜索时代的新基建

### 2.1 为什么 GEO 比 SEO 更重要

**传统 SEO：** 百度/Google 排名 → 点击 → 转化
**GEO（Generative Engine Optimization）：** Perplexity/ChatGPT/豆包 → 直接回答 → 被引用 → 信任 → 转化

AI 搜索引擎不再只是索引网页，而是**综合多个权威来源生成答案**。荣旺要成为被 AI 引用的权威来源。

### 2.2 GEO 覆盖矩阵

| 层级 | 目标 | 方法 |
|------|------|------|
| **L1 权威来源** | Wikipedia 词条 | 创建/补充：AKK菌、NADH、牛樟芝、灵芝孢子油 |
| **L1 权威来源** | 知乎问答 | 高赞回答，带来源链接 |
| **L2 结构化数据** | Schema.org | Article、Product、FAQ、Organization 全覆盖 |
| **L2 引用来源** | 权威健康媒体 | 与营养学期刊、健康媒体报道合作 |
| **L3 社交验证** | Reddit/Quora | 英文市场布局，海外用户引荐 |

### 2.3 Wikipedia 占位计划

**创建新词条（中文）：**
- AKK菌 (Akkermansia muciniphila) — 当前无完善中文条目

### 2.4 Schema.org 强化部署

所有页面必须包含以下结构化数据：

```typescript

// 健康内容（谨慎使用）

```

## 六、AI 内容生产管道

```

├── 工具：`duckduckgo-search` 搜索趋势

├── 输出：文章结构 + 要点 + SEO 建议
└── 参考：已上传至 `chroma` 的历史文章

├── 基础：产品知识库（`chroma` 向量库）

Step 4: 专家审核
├── 营养师审核功效描述
├── 合规团队审核监管风险

Step 5: SEO 优化
├── Schema 标记
├── 关键词密度优化
├── 内链 + 外链
└── meta description

Step 6: 多格式分发
├── 长文 → 网站文章
├── 摘要 → 知乎回答
├── 要点 → 小红书笔记
├── 脚本 → 抖音视频

Step 7: 效果追踪
```
