> "Turn one piece of content into many. The best social content isn't created from scratch — it's extracted from longer-form pillar content."

### 6.3 内容原子化原则

来自 `marketingskills/social-content`：

> "Each clip needs context; don't assume the viewer heard the rest"
> "Each post must work without context"

- ✅ 有独立的价值（不依赖原文也能理解）
- ✅ 有明确的 CTA（引导下一步行动）

## 七、AI 内容生产工作流

### 7.1 内容生产标准流程（SOP）

```
Step 1: 选题 & Brief（30分钟）
├── 从本月日历选择主题
├── 填写 Content Brief 模板
├── 确定目标关键词 + 搜索意图
└── 参考竞品内容（用 scrapling 抓取）

Step 2: 素材收集（30-60分钟）
├── 用 arxiv 搜索相关科研论文
├── 整理竞品内容要点（scrapling）
├── 收集用户评价/问答（了解真实痛点）
└── 输出：素材包（笔记 + 链接 + 数据）

Step 3: 初稿生成（60-90分钟）
├── 使用 ChatGPT/Claude 基于 Brief 生成
├── 参考素材包 + 产品知识库（chroma）
├── 对照 copywriting 原则检查
└── 输出：初稿 v1

Step 4: 专家审核（30分钟）
├── 营养师审核功效描述准确性
├── 合规审核（不能声称治病）
├── 编辑审核可读性
└── 输出：终稿

Step 5: SEO 优化（30分钟）
├── 添加内链（到相关产品页）
├── 添加 Schema（Article + FAQPage）
├── 优化 meta title + description
└── 输出：发布版

Step 6: 多格式拆解（60分钟）
├── 拆解为：知乎 + 微信 + 抖音 + 小红书 + X
├── 每个格式调整语气 + 长度
├── 配套视觉素材（封面图/字幕）
└── 输出：多格式内容包

Step 7: 发布 & 记录（30分钟）
├── 按日历发布到各平台
├── 记录发布链接 + 数据
└── 更新内容资产库（chroma）
```

**单篇长文总工时：4-5 小时**

### 标准周（适合 1 个写手 + AI 辅助）

```
周一（3小时）
├── 上午：选题 Brief（30分钟）
├── 上午：素材收集（60分钟）
└── 下午：初稿生成（90分钟）

周二（2小时）
├── 上午：初稿修改 + SEO优化（60分钟）
└── 下午：Schema添加 + 发布（60分钟）

周三（2小时）
├── 上午：内容复用 — 知乎 + 微信（60分钟）
└── 下午：内容复用 — 小红书 × 2（60分钟）

周四（2小时）
├── 上午：内容复用 — 抖音脚本 × 2（60分钟）
└── 下午：X/Twitter 线程（60分钟）

周五（1.5小时）
├── 上午：数据记录 + 内容库更新（30分钟）
├── 下午：下周选题 Brief（60分钟）
└── 全周：随时回复评论/私信
```

**周产能：1篇长文 + 6-8个社交内容单元**

### 冲刺周（适合 2 个写手或 AI 满负荷）

```
周一-周二：2篇长文（各3小时）
周三：1篇长文（3小时）
周四-周五：拆解 + 发布（4小时/天）
```

**冲刺周产能：3篇长文 + 15+个社交内容单元**

## 十、内容质量检查清单

### 发布前检查（每个内容单元）

```
SEO 检查
□ Title 包含目标关键词
□ Meta description 包含关键词 + CTA
□ 首段包含目标关键词
□ H2 标题反映搜索意图
□ 关键词自然出现，不堆砌
□ 内链 ≥ 2 个
□ 外链 → 权威来源（arxiv/PubMed）
□ 字数达标（≥1500字）

E-E-A-T 检查
□ 有署名（真实作者名）
□ 有"最后更新"日期
□ 有数据/研究引用（含来源）
□ 内容具体，非泛泛而谈
□ 无绝对化表述（"最有效"/"第一"）

合规检查
□ 无医疗声明（不能说"治疗XX病"）
□ 无伪科学表述
□ 产品功效有依据
□ 保健品应有"有助/有助于/可能有助于"措辞

技术检查
□ 图片有 alt 文字
□ URL 包含关键词
□ Canonical 正确
□ Schema markup 完整（Article + FAQ）
```
